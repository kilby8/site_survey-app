# SSO Authorization Endpoint - Action Required for Raymond

## Issue
The mobile app is getting a **404 error** when trying to access:
```
https://solarpro.solutions/api/auth/authorize
```

This endpoint is missing or not implemented on the SolarPro backend.

---

## What Needs to Be Done

Create a new endpoint on the SolarPro backend:

### Endpoint Details
- **URL**: `https://solarpro.solutions/api/auth/authorize` (or equivalent dev URL)
- **Method**: `GET`
- **Query Parameters**:
  - `redirect_uri` - URL to redirect back to after login (e.g., `https://site-survey-api-bpyz.onrender.com/auth/callback`)
  - `state` - CSRF protection nonce (random string generated by mobile app)

### What It Should Do
1. **Verify** that the user is logged into SolarPro (session check)
2. **If not logged in**: Show SolarPro login form
3. **If logged in**: 
   - Get the user's SolarPro credentials (user_id, email, name, org_id, etc.)
   - Generate a **JWT signed with `SOLARPRO_HANDOFF_SECRET`** containing:
     ```json
     {
       "solarpro_user_id": "user-123",
       "solarpro_email": "user@company.com",
       "solarpro_name": "John Doe",
       "solarpro_org_id": "org-456",
       "email": "user@company.com",
       "name": "John Doe",
       "jti": "unique-uuid-here",
       "exp": 1234567890,
       "iat": 1234567800
     }
     ```
   - **Redirect** the browser to: `{redirect_uri}?token={JWT}&state={state}`

---

## Example Authorization Flow

```
Mobile App
  ↓ opens browser
  https://solarpro.solutions/api/auth/authorize
  ?redirect_uri=https://site-survey-api.onrender.com/auth/callback
  &state=abc123xyz
  
User logs in on SolarPro
  ↓
SolarPro backend generates JWT
  ↓
SolarPro redirects browser to:
  https://site-survey-api.onrender.com/auth/callback
  ?token=<JWT>
  &state=abc123xyz
  
Our backend deeplinks back to app:
  sitesurvey://login?token=<JWT>&state=abc123xyz
  
Mobile app receives token via callback listener
  ↓
Mobile app exchanges token via POST /api/users/solarpro-sso
  ↓
User authenticated ✅
```

---

## Critical: SOLARPRO_HANDOFF_SECRET Must Match

**IMPORTANT**: The JWT **MUST** be signed with this secret:
```
SOLARPRO_HANDOFF_SECRET = prod_handoff_secret_2026_rotate_me
```

This same secret is configured in:
- **Our backend** (Render): `site-survey-api` service env var
- **SolarPro backend**: Must be set to the SAME value

If the secrets don't match:
- Our backend will reject the JWT with `401 Unauthorized`
- Mobile app login will fail
- Users cannot access the app

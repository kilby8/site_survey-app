Team,

We are ready for staging webhook traffic.

Webhook receiver endpoint
- Method: POST
- URL: https://solar-pro.app/api/webhooks/survey-complete

Current receiver mode
- Pre-ingest accept mode is enabled.
- Valid signed events return HTTP 202 with:
  {"ok":true,"code":"ACCEPTED_PRE_INGEST","event_id":"..."}
- Duplicate event IDs return HTTP 200 with:
  {"ok":true,"duplicate":true,"event_id":"..."}

Headers required
- X-Survey-Signature
- X-Survey-Timestamp
- X-Survey-Event-Id

Signature algorithm (exact)
- HMAC-SHA256 over: ${timestamp}.${rawBody}
- Signature header format: sha256=<lowercase_hex_digest>

Body contract (thin event)
{
  "event": "survey.completed",
  "event_id": "uuid",
  "occurred_at": "ISO-8601 timestamp",
  "survey_id": "uuid",
  "status": "submitted",
  "completed_at": "ISO-8601 timestamp"
}

Replay/idempotency behavior
- event_id is idempotent; duplicate deliveries are accepted and reported as duplicate.

Selected integration decisions (locked)
1) Full survey fetch auth: Bearer token
2) Photo format: /uploads/... URLs with bearer auth
3) Handoff launch URL: /new-survey?token=...

Required secrets (out-of-band exchange)
1) SURVEY_WEBHOOK_SECRET
2) SOLARPRO_HANDOFF_SECRET

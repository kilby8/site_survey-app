## Problem


---

## Solution: Create This Endpoint

### Route
```
GET /api/auth/authorize
```

### Query Parameters
- `redirect_uri` - Where to send the browser after login (e.g., https://site-survey-api-bpyz.onrender.com/auth/callback)
- `state` - CSRF nonce (pass through unchanged)

### Implementation Checklist

- [ ] Check if user is logged into SolarPro
- [ ] If NOT logged in → show login form
- [ ] If logged in → proceed to step below
- [ ] Create JWT with claims:
  ```json
  {
    "solarpro_user_id": "their-user-id",
    "solarpro_email": "their@email.com",
    "solarpro_name": "Their Name",
    "solarpro_org_id": "their-org-id",
    "jti": "uuid-v4",
    "exp": <unix-timestamp-1-hour-from-now>,
    "iat": <current-unix-timestamp>
  }
  ```
- [ ] **Sign JWT with this Secret:**
  ```
  prod_handoff_secret_2026_rotate_me
  ```
  (Must be HS256 algorithm)
- [ ] Redirect browser to:
  ```
  {redirect_uri}?token={JWT}&state={state}
  ```

---

## Why This Works

1. Mobile app → Opens authorize URL
2. User logs in on SolarPro
3. SolarPro signs JWT and redirects
4. Our backend receives callback
5. Our backend deeplinks back to mobile app
6. Mobile app exchanges JWT for access token
7. User authenticated ✅
